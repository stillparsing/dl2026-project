This section abstracts out the common features of these commands that serve as a requirement for an
interface protocol to implement the present specification.  These sections address communications on
protocols 0x01 and 0x02 only (see Table 25).
The following assumptions are made regarding the interface commands:
a. The interface commands have two parts: (1) a command block and (2) a data payload.
Each host interface protocol has its own minimum payload size. The payload size is not
related to the 'logical block size' of the user data on the medium of the Storage Device. See
the definitions of IF-SEND and IF-RECV for details.
b. There is at least one command in the interface protocol that transfers data from the host to
the Storage Device.  These commands are called IF-SEND.
c. There is at least one command in the interface protocol that transfers data from the Storage
Device to the host.  These commands are called IF-RECV.
The abstracted command block of the interface commands are described in the format defined in Table
24.
The mapping of the IF-SEND and IF-RECV commands to specific interface protocol commands are
described in [2].

Table 24 Interface Command – Command Block
```markdown
| Field           | Description                                                   |
|-----------------|---------------------------------------------------------------|
| Command         | Either IF-SEND or IF-RECV.                                   |
| Protocol ID     | Between 0x01 and 0x06 (see Table 25)                         |
| Transfer Length | at least 2 bytes (the length of this field varies by host interface) The amount of data to be transferred. |
| ComID          | The ComID to be used, for Protocol IDs 0x01, 0x02, 0x06 (see 3.3.2 and 3.3.3). |
```
