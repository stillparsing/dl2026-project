5.8.2.1 Log (Object Table)
Log tables are object tables that store log entries.  Each row in a Log table is an entry.
There MAY be more than one Log table in an SP.  Each of these Log tables SHALL have a unique
name.  Only the default Log table, which has the name “Log”, stores System log entries.  The Log table
described in this section acts as the template for all log tables.  Log tables are created using the
CreateLog method.  User-created log tables each have an associated row in the LogList table.
Log tables SHALL only be accessible via table level methods. Individual log table rows SHALL NOT
have associated rows in the AccessControl table.
